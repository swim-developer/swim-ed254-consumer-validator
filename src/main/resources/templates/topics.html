Provider Exception Types

providerExceptions messages signal AMAN system status changes:

Exception Description Impact
AMAN UnavailableAMAN system is completely downNo arrival sequences published; upstream controllers revert to procedural control
Degraded ModeAMAN running with reduced capabilitySequences may be less accurate; advisory information may be unavailable
Sequencing DisabledSequencing temporarily suspended for an aerodromeFlights not assigned target times; controllers manage spacing manually

Subscription Filters

When creating a subscription, the consumer can specify filters to receive only relevant messages:

Filter Type Description
aerodromeString (ICAO code)Filter by destination aerodrome (e.g., LFPG, EGLL)
messageTypeStringFilter by message type: arrivalSequence or providerExceptions
includeSupplementaryDataBooleanInclude extended advisory information in sequence entries
Operational Context: Extended AMAN (E-AMAN) extends arrival sequencing from the traditional 40-100 NM to 150-350 NM from the destination airport. This enables upstream ANSPs to issue speed and route advisories to flights still in cruise, reducing fuel burn and airborne holding. The ED-254 standard (EUROCAE, June 2018) defines the service interface bindings for cross-border E-AMAN operations under CP1 (EU Regulation 2021/116).
